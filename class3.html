Mongoose

Mongoose provides mongoDB object modeling that can do things like validation and default values. Open the file models/dinosaur.js. Our dinosaur model is very simple:


var mongoose = require('mongoose');
var Schema = mongoose.Schema;
var DinosaurSchema = new Schema({
  name: String
});

module.exports = mongoose.model('Dinosaur', DinosaurSchema);

Mongo Ids

If you've used a framework like Ruby on Rails before you may have seen that objects in a SQL database get assigned ids starting at 1. Mongo assigns unique ids in a different manner, as you can see by the long string of numbers and letters.

Mongoose

Add a default value:


var DinosaurSchema = new Schema({
  name: {
    type: String,
    default: 'Dino'
  },
  age: Number
});

Available schema types

  • String
  • Number
  • Date
  • Buffer
  • Boolean
  • Mixed
  • ObjectId
  • Array

Add instance methods


DinosaurSchema.methods.speak = function() {
  var greeting = "My name is " + this.name + " and my age is " + this.age;
  console.log(greeting);
  return greeting;
}

back to server.js

Now we can call the speak() method for one of our dinos.


app.get('/dinosaurs/:id', function(req, res) {
  Dinosaur.findById(req.params.id, function(err, dino) {
    var greeting = dino.speak();
    res.render('read', {
      dinosaur: dino,
      greeting: greeting
    });
  });
});

Hooks

We can also add hooks to our model that run on find, save, remove, initialization, and validate.


DinosaurSchema.post('save', function(doc) {
  console.log('Dino with id %s has been saved', doc._id);
});

Mongoose Queries

We use a few queries in our API to find one or many dinos:


Dinosaur.find();
Dinosaur.findById();

Calling functions

There are two ways to call mongo queries:


Dinosaur.find(callback);
                

Dinosaur.find().exec(callback);

More queries


// Find one named Beyonce
Dinosaur.findOne({name: 'Beyonce'}).exec(callback);
// Find all named Beyonce
Dinosaur.find({name: 'Beyonce'}).exec(callback);

Advanced queries

What if we want to find only the first 3 teenage dinos sorted by age? One plus of using .exec() is the ability to chain our query:


Dinosaur.find({
    age: { $gt: 12, $lt: 20 }
}).limit(3).sort({ age: -1 }).exec(callback);

Query builder

If you prefer you can use another syntax:


Dinosaur.
  find().
  where('name').equals('Beyonce').
  where('age').gt(12).lt(20).
  limit(10).
  sort('-name').
  select('name').
  exec(callback);

.select()

Using .select() on your query allows you to specify which fields you want returned:


Dinosaur.
  find().
  limit(10).
  select('name').
  exec(callback);

Grunt

Grunt is a task runner that can perform tasks like minification, compilation, unit testing, linting. Globally install the Grunt command line interface (CLI) and create a Gruntfile.js:


npm install -g grunt-cli
touch Gruntfile.js

Dependencies

In your package.json add these as dev dependencies:


"devDependencies": {
    "grunt": "~0.4.5",
    "grunt-contrib-clean": "*",
    "grunt-htmlhint": "*",
    "grunt-contrib-uglify": "~0.5.0",
    "matchdep": "*"
}

Then run `npm install`

Create a Grunt task


module.exports = function(grunt){

    grunt.initConfig({
        pkg: grunt.file.readJSON('package.json')
    });

    grunt.registerTask('default', []);

};

Grunt

Above grunt.initConfig() add this:


require("matchdep").filterDev("grunt-*").forEach(grunt.loadNpmTasks);

The matchdep module will install any devDependencies from the package.json that begin with "grunt-"

Watch tasks

Inside grunt.initConfig add these options:

htmlhint: {
    build: {
        options: {
            'tag-pair': true,
            'tagname-lowercase': true,
            'attr-lowercase': true,
            'attr-value-double-quotes': true,
            'spec-char-escape': true
        },
        src: ['index.html']
    }
},
watch: {
    html: {
        files: ['index.html'],
        tasks: ['htmlhint']
    }
}

Watch tasks

Now open another terminal and run:


grunt watch

When you edit your HTML file this watch task will run and validate the HTML.

Uglify your code

Have you ever seen minified Javascript? Uglify is a module that compresses your code and is used a lot for production applications. Add this option to grunt.initConfig:


uglify: {
    build: {
        files: {
            'build/js/base.min.js': ['base.js']
        }
    }
},
watch: {
    js: {
        files: ['base.js'],
        tasks: ['uglify']
    },
    html: {...}
}

Check out what other options Grunt can do for you!
